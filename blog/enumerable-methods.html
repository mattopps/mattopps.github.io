Hello Again Blog!

Well it basically took me until yesterday to even grasp the basic definition and use of an Enumerable. Every time the docs said "returns an enumerator" I was thoroughly confused. This was especially so because repl/irb tests which specifically test those methods with no clocks passed return the actual word "Enumerator", followed by the input. I was expecting some other word, i.e. a type of Enumerator.

Basically Enumerable is another way of referring to a group of methods that work on any object type that is a collection: Sets, Ranges, Arrays, and Hashes. Enumerables are are a group of methods that those classes all have access to because they all have an each method. Enumerables in many cases are supercharged and wildly powerful each methods that iterate through the collection and perform an operation on each object. Technically I think Enumerable is a module. But I'm less than 50% sure what modules are at this point to how to implement them

Cycling

I chose to look at the cycle enumerable for many reasons. I like cycling, it was first, and I spent a lot of time playing with it in repl/irb so that I felt committed to it at that point.

Basically calling cycle is like calling each multiple times. Though calling each returns the original source unaltered while cycle returns nil. That is if it returns anything at all. Because calling the cycle method with any number less than 1 as an argument (negative numbers or zero) will have no effect. It will literally do nothing. It will be as if you did not run anything.

Some oddities I find in this method. It doesn't require an argument, but if you run it without an argument it will repeat forever. Which Doesn't seem like anybody would ever want to do, so I think it's odd that it doesn't raise an argument error if no arguments are called.

So let's say you had a hash h =[2,4,6,8] and you tried to run the following line print h.cycle(5) {|num| num*num} You will get nil and no output to the console because the value of everything after print amounts to nil. Running h.cycle(5) {|num| print num*num} will output the squares of each index value, in this case 4163664 five times. It will still return nil. Cycle always returns nil if it runs fully.
